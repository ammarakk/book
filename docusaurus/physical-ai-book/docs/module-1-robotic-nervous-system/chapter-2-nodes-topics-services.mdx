# Chapter 2: Nodes, Topics, Services

This chapter explores the fundamental communication patterns in ROS 2: nodes, topics, and services.

## Overview

In ROS 2, communication between different parts of your robot system happens through three main patterns: topics (for streaming data), services (for request/reply interactions), and actions (for goal-oriented interactions).

## Learning Objectives

By the end of this chapter, you will understand:
- How nodes communicate with each other
- The differences between topics, services, and actions
- When to use each communication pattern

## Nodes

A node is a process that performs computation. Nodes are the fundamental building blocks of a ROS 2 program. Multiple nodes are managed in a graph and communicate with each other using topics, services, and actions.

### Creating a Node

```python
import rclpy
from rclpy.node import Node

class MyNode(Node):
    def __init__(self):
        super().__init__('my_node')
        self.get_logger().info('MyNode has been started')

def main(args=None):
    rclpy.init(args=args)
    node = MyNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```

## Topics

Topics enable asynchronous message passing between nodes. Publishers send messages to a topic, and subscribers receive messages from a topic. This is a many-to-many relationship.

### Publisher Example

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalPublisher(Node):

    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = 'Hello World: %d' % self.i
        self.publisher_.publish(msg)
        self.get_logger().info('Publishing: "%s"' % msg.data)
        self.i += 1
```

### Subscriber Example

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MinimalSubscriber(Node):

    def __init__(self):
        super().__init__('minimal_subscriber')
        self.subscription = self.create_subscription(
            String,
            'topic',
            self.listener_callback,
            10)
        self.subscription  # prevent unused variable warning

    def listener_callback(self, msg):
        self.get_logger().info('I heard: "%s"' % msg.data)
```

## Services

Services provide a request/reply interaction pattern. A client sends a request to a service server, which processes the request and returns a response.

### Service Server Example

```python
from add_two_ints_srv.srv import AddTwoInts
import rclpy
from rclpy.node import Node

class MinimalService(Node):

    def __init__(self):
        super().__init__('minimal_service')
        self.srv = self.create_service(AddTwoInts, 'add_two_ints', self.add_two_ints_callback)

    def add_two_ints_callback(self, request, response):
        response.sum = request.a + request.b
        self.get_logger().info('Incoming request\na: %d b: %d' % (request.a, request.b))
        return response
```

### Service Client Example

```python
from add_two_ints_srv.srv import AddTwoInts
import rclpy
from rclpy.node import Node

class MinimalClientAsync(Node):

    def __init__(self):
        super().__init__('minimal_client_async')
        self.cli = self.create_client(AddTwoInts, 'add_two_ints')
        while not self.cli.wait_for_service(timeout_sec=1.0):
            self.get_logger().info('service not available, waiting again...')
        self.req = AddTwoInts.Request()

    def send_request(self, a, b):
        self.req.a = a
        self.req.b = b
        self.future = self.cli.call_async(self.req)
        rclpy.spin_until_future_complete(self, self.future)
        return self.future.result()
```

## Actions

Actions are used for goal-oriented tasks that may take a long time to complete. They provide feedback during execution and can be canceled.

## Practical Examples

Here's a practical example combining all three communication patterns:

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String
from example_interfaces.srv import AddTwoInts

class RobotController(Node):
    def __init__(self):
        super().__init__('robot_controller')
        
        # Topic publisher for robot status
        self.status_publisher = self.create_publisher(String, 'robot_status', 10)
        
        # Service server for robot commands
        self.command_service = self.create_service(
            AddTwoInts, 
            'robot_command', 
            self.command_callback
        )
        
        # Timer to periodically publish status
        self.timer = self.create_timer(1.0, self.publish_status)
    
    def command_callback(self, request, response):
        # Process the command
        response.sum = request.a + request.b  # Example processing
        self.get_logger().info(f'Command received: {request.a} + {request.b} = {response.sum}')
        return response
    
    def publish_status(self):
        msg = String()
        msg.data = 'Robot is operational'
        self.status_publisher.publish(msg)
```

## Summary

Understanding the different communication patterns in ROS 2 is crucial for designing effective robotic systems. Topics are best for streaming data, services for request/reply interactions, and actions for goal-oriented tasks.
